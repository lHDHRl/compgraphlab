# Задание 8.3. Собственный формат (YCrCb)

## Описание

Реализация собственного формата хранения изображений на основе цветового пространства YCrCb с субдискретизацией и сжатием.

## Теория

### RGB → YCrCb

$$Y = 0.299R + 0.587G + 0.114B$$
$$Cr = 128 + (R - Y) \cdot 0.713$$
$$Cb = 128 + (B - Y) \cdot 0.564$$

### YCrCb → RGB

$$R = Y + 1.403 \cdot (Cr - 128)$$
$$G = Y - 0.344 \cdot (Cb - 128) - 0.714 \cdot (Cr - 128)$$
$$B = Y + 1.773 \cdot (Cb - 128)$$

### Субдискретизация

- **4:4:4** — без сжатия, все каналы полного разрешения
- **4:2:2** — $Cr$ и $Cb$ сжаты в 2 раза по горизонтали (среднее пар пикселей):

$$Cr'_i = \frac{Cr_{2i} + Cr_{2i+1}}{2}$$

- **4:2:0** — $Cr$ и $Cb$ сжаты в 2 раза по обеим осям (среднее блоков $2 \times 2$):

$$Cr'_{i,j} = \frac{Cr_{2i,2j} + Cr_{2i+1,2j} + Cr_{2i,2j+1} + Cr_{2i+1,2j+1}}{4}$$

### RLE (Run-Length Encoding)

Последовательность пар $(\text{значение}, \text{длина серии})$. Эффективно для однородных областей.

## Алгоритм

1. Загрузить изображение, преобразовать RGB → YCrCb
2. Применить субдискретизацию к каналам $Cr$, $Cb$
3. (Опционально) применить RLE-сжатие
4. Сохранить в собственный формат / загрузить обратно
5. Восстановить: декодировать RLE → восстановить разрешение $Cr$, $Cb$ → YCrCb → RGB
